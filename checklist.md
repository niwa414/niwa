下面给你一份**工程化 checklist**，格式是“功能点 → Helion 公开证据 → 你应当交付的最小可复现实验（MVP）→ 诊断量/验收口径”。
---

## 使用方法：先固化 5 个“门槛算例”，再扩展

建议你把回归体系先收敛到 5 个算例（每个算例都能一键跑 + 一键出图 + 一键判定 PASS/FAIL）：

1. 2D Hall-MHD：FRC merging（合并）
2. 2D Hall-MHD：compression（压缩，时变外加磁场/驱动）
3. 稳定性：tilt（m=1）增长率提取（准 3D 或真 3D）
4. Hybrid：MHD ↔ Hybrid 的一致性对照（同一初值/同一驱动下关键量对齐）
5. 工程耦合：外部电路（动态 circuits）与等离子体的耦合演示（闭环，而非仅“波形驱动”）

---

## Checklist A：2D 轴对称 Hall-MHD（CYGNUS 对标域）

### A1. 2D 轴对称 Hall-MHD 主方程 + 数值稳定推进

* Helion 证据：CYGNUS 被描述为 2D 轴对称 Hall-MHD，并强调为工程应用进行了推进与稳健性方面的实现（含半隐式推进等）。
* 你应交付的最小实验：

  * 2D R–Z：给定平衡/初值，开 Hall 项、关 Hall 项各跑一遍，比较色散/结构变化趋势（不是追求“对错”，而是确保实现一致、可控）。
* 验收口径（建议固定）：

  * CFL 稳定范围、能量收支漂移、∇·B 误差上限、Hall 项开/关的差异性可解释（例如更强的细结构/更快的重联特征）。

### A2. FRC 过程段覆盖：formation / translation / merging / compression（至少把你要对标的段做实）

* Helion 证据：CYGNUS 明确覆盖 formation、translation、merging、compression。
* 你应交付的最小实验（按段拆）：

  * merging：双 FRC（或双等离子团块）合并，给出合并时间、磁通变化、峰值压力/密度变化。
  * compression：在预置/时变外磁场下压缩，给出压缩比、磁通保留率、温度/压力增幅。
* 验收口径（建议固定）：

  * “合并质量指标”（例如 trapped flux 保留率、混合层厚度、峰值 β、等效电流层宽度）；
  * “压缩质量指标”（压缩比、磁通损失率、能量输入与内能增量的对应关系）。

### A3. 动态外部电路耦合（闭环）

* Helion 证据：CYGNUS 支持静态与动态外部电路与 MHD 方程耦合（这是论文级表述）。
* 你应交付的最小实验：

  * 用一个最简单的 RLC（或等效驱动器）模型与线圈边界/源项联立推进：电流 I(t) 不再是预先给定曲线，而是受等离子体“等效电感/反电动势/电阻”反馈影响。
* 验收口径：

  * 对同一电路参数，给出“有/无等离子体反馈”的 I(t)、V(t) 差异；
  * 证明耦合是数值稳定的（步长敏感性、误差随步长收敛）。

### A4. 任意轴对称几何（cut-cell / embedded boundary）

* Helion 证据：CYGNUS 论文明确提到对任意轴对称几何的 cut-cell 支持。
* 你应交付的最小实验：

  * 至少一个“非规则轴对称腔体/内壁”的算例（哪怕是简单的锥形/台阶），比较与规则圆柱的差异。
* 验收口径：

  * 质量/磁通守恒误差不因几何复杂而失控；
  * 壁面通量条件严格满足（无非物理泄漏）。

## Checklist B：3D（或准 3D）Hybrid（离子动理学 + 电子流体）PIC（WarpX/HYM 对标域）

### B1. Hybrid 物理模型：离子 PIC + 电子流体（广义欧姆定律框架）

* Helion 证据：Helion 披露其在 WarpX 上做 3D hybrid-kinetic 建模；WarpX 文档也明确 Hybrid-PIC 的理论基座（广义欧姆定律、电子流体、离子动理学）。
* 你应交付的最小实验：

  * 从同一个流体初值出发，跑 Hybrid，输出关键量（离子分布函数特征、J、E、B、电子压力项贡献等）。
* 验收口径：

  * 与 MHD 的大尺度量一致（例如整体磁通/总体动量趋势），但能体现动理学差异（例如非麦氏尾部、各向异性压力、微不稳定迹象）。

### B2. 3D tilt（m=1）不稳定性：合并后 tilt 演化

* Helion 证据：Helion 官方披露了“3D hybrid-PIC 合并后出现 tilt”的公开视频片段；APS DPP 也有面向 Polaris 的 3D hybrid 摘要。
* 你应交付的最小实验：

  * 准 3D（RZ + modes）或真 3D：给一个可控的 m=1 扰动，输出 tilt 振幅随时间曲线并拟合增长率。
* 验收口径：

  * 增长率（线性阶段斜率）可重复（对随机种子/粒子数/网格有收敛趋势）；
  * 饱和阶段形态合理（不要求与 Helion 一致，但要求自洽且可解释）。

### B3. 电子能量方程增强（Helion 披露的“particle-based electron energy equation”）

* Helion 证据：Helion 明确写到在现有 solver 中加入“基于粒子的电子能量方程”。
* 你应交付的最小实验：

  * 若你不实现同款“particle-based”，至少要给出“电子能量演化”更完整的闭合：Te 的时空演化、热通量/加热项、与压缩/重联的能量转换一致。
* 验收口径：

  * 电子能量收支闭合（输入功、欧姆加热、压缩功、边界通量）；
  * 对压缩算例，Te 的增长与压缩比有合理标度。

### B4. 压缩大动态范围数值策略（density floor、field sub-stepping、时变外场等）

* Helion 证据：APS DPP 2025 摘要披露了在 3D hybrid kinetic PIC 压缩仿真中采用 density floor scaling、field sub-stepping、时变外加磁场等来处理动态范围。
* 你应交付的最小实验：

  * 同一压缩算例：对比“不开这些策略” vs “开策略”的可跑性与物理量稳定性。
* 验收口径：

  * 能跑到目标压缩比；
  * 关键守恒量（总能量/磁通）没有被策略引入的数值伪影主导（需要量化差异）。

### B5. MHD ↔ Hybrid 的对照研究（HYM 风格的“同题对照”）

* Helion 证据：HYM 相关公开材料（INFUSE、arXiv/会议摘要）强调用 hybrid 与 MHD 对照研究 FRC acceleration/merging/compression 与稳定性。
* 你应交付的最小实验：

  * 同一初值/同一驱动：MHD（或 Hall-MHD）跑到某一时刻导出场量，再用 Hybrid 接续；或两边并行从同初值起跑，对齐相同时刻的宏观量。
* 验收口径：

  * 至少 3 个宏观量对齐：例如轴向长度、峰值密度、总磁能/内能、trapped flux；
  * 指出差异来源（动理学 vs 数值 vs 边界/闭合）。

## Checklist C：基准、验证与“可引用证据链”（把工程成果变成论文式证据）

### C1. 历史算例/实验基准（benchmark）与诊断对齐

* Helion 证据：公开披露中提到对历史实验（例如 LSX）进行 benchmark，并把仿真用于指导线圈时序改进并得到实验反馈。
* 你应交付的最小实验包（建议命名为 evidence pack）：

  * 固定版本号（commit）、固定输入卡、固定诊断脚本、固定输出摘要（1 页 PDF 或 markdown）。
* 验收口径：

  * 每个算例至少输出：守恒量曲线、磁通曲线、主要不稳定模态振幅、关键截面图、运行成本（步数、粒子数、GPU/CPU 时间）。

### C2. “对外可引用”的最小素材清单（你写报告/论文时最值钱的）

* 每个门槛算例至少要有：

  1. 输入与参数表（含几何、驱动、电阻/碰撞模型、Hall/闭合参数）
  2. 3–5 个核心诊断图（统一坐标轴与单位）
  3. 一句话结论 + PASS/FAIL 规则
  4. 与 Helion 公布口径的对应关系（例如：tilt、compression 稳定性、动态范围处理、电路耦合）
